## מבוא
מערכת זו נועדה לנהל את שרשרת האספקה והרכש של גן חיות מודרני. המערכת מנהלת מעקב אחר ספקים, קטלוג מוצרים מגוון (מזון לחיות, ציוד רפואי, חומרי תחזוקה והעשרה), הזמנות רכש, חשבוניות ותשלומים. המטרה היא לייעל את תהליך ההזמנות מול ספקים שונים ולשמור על היסטוריית תנועות מלאה של 20,000 הזמנות רכש לאורך תקופה ארוכה.

## שיטות אכלוס נתונים
כדי לדמות מערכת חיה ואמיתית, אכלסנו את מסד הנתונים בלמעלה מ-42,000 רשומות, תוך שימוש ב-3 שיטות שונות על פי דרישות הפרויקט:

**1. שימוש בכלי חיצוני (Mockaroo) - ייצור ספקים:**
שימוש באתר Mockaroo ליצירת 500 ספקים פיקטיביים עם פרטי קשר מלאים (שמות, קטגוריות, טלפונים).
![Mockaroo](images/image-2.png)

**2 ו-3. שימוש בתכנות (Python) ופקודות INSERT מפורשות:**
פיתוח סקריפט Python חכם שמייצר שמות מוצרים ריאליסטיים מעולם התוכן של גני חיות (ללא כפילויות), ומחולל פקודות SQL ישירות. הסקריפט יצר 20,000 הזמנות רכש וכ-20,000 שורות הזמנה, לצד מאות חשבוניות ותשלומים בהלימה מושלמת של המפתחות הזרים.

## גיבוי ושחזור
בהתאם לדרישות הפרויקט, ביצענו גיבוי מלא לבסיס הנתונים בשתי שיטות נפרדות. הגיבויים נבדקו ויכולים לשחזר את המערכת בשלמותה על כל מחשב.

**שיטה 1: דרך ממשק המשתמש (pgAdmin UI):**
ביצוע גיבוי ויזואלי דרך סביבת הניהול pgAdmin אל קובץ `backup_UI.sql`.
![UI Backup](images/image-6.png)

**שיטה 2: דרך שורת הפקודה (Terminal / Docker CLI):**
התחברות ישירה לקונטיינר ה-PostgreSQL בעזרת פקודות Docker, ושאיבת הנתונים באמצעות הפקודה `pg_dump` לקובץ `backup_CLI.sql`.
![CLI Backup](images/image-5.png)
